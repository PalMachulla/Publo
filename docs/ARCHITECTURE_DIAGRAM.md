## ğŸ¯ Key Design Decisions

### 1. Two-Phase Orchestration
**Why:** Node must exist before agents can save content to it.
**How:** First orchestration creates structure, second generates content.
**Benefit:** Clean separation of concerns, reliable persistence.

### 2. WorldState Update
**Why:** Agents need access to fresh node data.
**How:** Call `worldState.setActiveDocument()` after node creation.
**Benefit:** Agents always have current state, no stale data.

### 3. Action Filtering
**Why:** Can't execute content actions without node ID.
**How:** Check `hasNodeId` before routing to agents.
**Benefit:** Prevents errors, ensures proper sequencing.

### 4. Server-Side Persistence
**Why:** Client-side RLS restrictions prevent agent writes.
**How:** Use `/api/agent/save-content` with admin client.
**Benefit:** Bypasses RLS, ensures writes succeed.

### 5. DAG Execution
**Why:** Enable parallel content generation for multiple sections.
**How:** Build dependency graph, execute independent tasks in parallel.
**Benefit:** Faster execution, better resource utilization.
